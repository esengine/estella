Every scene renders through a camera — an entity with a `Camera` component. You
control zoom, follow a target, convert between screen and world space, shake the
view, and switch between multiple cameras.

## The `Camera` component

Key fields for a 2D game:

- `orthoSize` — the vertical half-height in world units: **this is your zoom**
  (smaller = zoomed in).
- `isActive` / `priority` — which camera is the active one when several exist
  (highest priority active camera wins).
- `viewport` (a `Vec4`, `x/y/w/h` in 0–1) — the screen rectangle the camera draws
  to, for split-screen or picture-in-picture.
- `clearFlags`, `pixelPerfect`, `aspectRatio`.

## Follow a target

Add a `FollowTarget` to the camera and point it at an entity. Usually you set this
in the editor; in code:

```typescript
import { defineSystem, Query, Commands, Camera, FollowTarget } from 'esengine';

const follow = defineSystem([Query(Camera), Commands()], (cameras, cmds) => {
  for (const [cameraEntity] of cameras) {
    cmds.entity(cameraEntity).insert(FollowTarget, {
      target: playerEntity,   // the Entity to follow (from your player Query)
      offsetY: 40,
      deadzone: 24,           // roam this far before the camera moves
      damping: 0.25,          // smoothing (0 = snap)
    });
  }
});
```

## Screen ↔ world

Read `CameraView` to convert coordinates — for placing entities under the cursor
or checking what's on screen:

```typescript
import { defineSystem, Res, CameraView, Input } from 'esengine';

const pick = defineSystem([Res(CameraView), Res(Input)], (view, input) => {
  const world  = view.screenToWorld(input.mouseX, input.mouseY);  // { x, y } | null
  const screen = view.worldToScreen(x, y);                        // { x, y } | null
  const mouse  = view.getWorldMousePosition();                    // { x, y } | null
  const bounds = view.getWorldBounds();                           // { left, right, bottom, top }
});
```

## Shake the camera

`shakeCamera` adds a decaying perturbation to the **rendered** view (never to the
Transform, so it always recovers):

```typescript
import { shakeCamera } from 'esengine';

shakeCamera(app, { amplitude: 12, rotation: 0, frequency: 22, duration: 0.4 });
```

## Multiple cameras

With several `Camera` entities, `isActive` + `priority` decide which is live. The
camera director can **blend** the view from one camera to another over time
(rather than cutting) — for cinematic hand-offs — and `viewport` lets two cameras
share the screen for split-screen play.

<Aside type="tip">
  Animate `orthoSize` with a [tween](/docs/guides/animation/)
  (`TweenTarget.CameraOrthoSize`) for a smooth zoom in/out.
</Aside>
