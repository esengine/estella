Estella's physics runs in the C++/WebAssembly core. You add the physics plugin,
attach a `RigidBody` and a collider to an entity, and drive or query it through
the `Physics` API.

## Enable physics

```typescript
import { PhysicsPlugin } from 'esengine/physics';

app.addPlugin(new PhysicsPlugin('physics.js', {
  gravity: { x: 0, y: -9.81 },
}));
```

The module loads asynchronously; physics is ready once its init settles.

## Bodies and colliders

Attach a `RigidBody` plus one collider shape. **Collider dimensions are in
meters, not pixels** â€” divide pixel sizes by your pixels-per-unit (PPU):

```typescript
import { RigidBody, BodyType, CircleCollider, BoxCollider } from 'esengine';

const PPU = 100;

cmds.spawn()
  .insert(Transform, { position: { x, y, z: 0 } })
  .insert(RigidBody, { bodyType: BodyType.Dynamic })
  .insert(CircleCollider, { radius: 24 / PPU, restitution: 0.4 });
```

- **`BodyType`** â€” `Static` (immovable), `Kinematic` (moved by you), `Dynamic`
  (moved by the simulation).
- **Collider shapes** â€” `BoxCollider`, `CircleCollider`, `CapsuleCollider`,
  `SegmentCollider`, `PolygonCollider`, `ChainCollider`. All share `density`,
  `friction`, `restitution`, `isSensor`, `categoryBits`, `maskBits`.
- **Joints** â€” `RevoluteJoint`, `DistanceJoint`, `PrismaticJoint`, `WeldJoint`,
  `WheelJoint`.

## Collision layers

`categoryBits` (what layer this body is on) and `maskBits` (what it collides
with) are 16-bit filters. Two bodies collide only if each one's category is in
the other's mask:

```typescript
const LAYER_PLAYER = 0x0001;
const LAYER_GROUND = 0x0008;

.insert(CircleCollider, {
  radius: 16 / PPU,
  categoryBits: LAYER_PLAYER,
  maskBits: LAYER_PLAYER | LAYER_GROUND,
});
```

## Forces, velocity, and queries

Read the `Physics` resource in a system to push bodies and query the world:

```typescript
import { defineSystem, Res, Physics } from 'esengine';

defineSystem([Res(Physics)], (physics) => {
  physics.applyImpulse(player, { x: 0, y: 5 });      // jump
  physics.setLinearVelocity(player, { x: 3, y: 0 });

  // Queries take pixel-space and auto-scale by PPU.
  const hits = physics.raycast(origin, direction, 200);
  const inRange = physics.overlapCircle(center, 64);
});
```

## Collision events

The `PhysicsEvents` resource carries this frame's contacts:

```typescript
import { PhysicsEvents } from 'esengine';

defineSystem([Res(PhysicsEvents)], (events) => {
  for (const c of events.collisionEnters) { /* c.entityA, c.entityB */ }
  for (const s of events.sensorEnters)   { /* trigger volumes */ }
});
```

<Aside type="caution">
  Set `isSensor: true` on a collider to get enter/exit events **without** a
  physical response (trigger zones). Physics steps at a fixed timestep, not the
  frame delta, so read `PhysicsEvents` every frame.
</Aside>
