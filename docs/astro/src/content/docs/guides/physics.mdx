Estella's physics runs in the C++/WebAssembly core (Box2D). You attach a
`RigidBody` and a collider to an entity, and drive or read it through the
physics resources.

## Enabling physics

Physics is a loadable module. In the editor and the web runtime it loads
automatically when a scene uses physics components тАФ you don't wire anything up.
In a custom app you can add it yourself:

```typescript
import { PhysicsPlugin } from 'esengine/physics';

app.addPlugin(new PhysicsPlugin('physics.js', {
  gravity: { x: 0, y: -9.81 },
}));
```

## Bodies and colliders

Attach a `RigidBody` plus one collider. **Collider dimensions are in meters, not
pixels** тАФ divide pixel sizes by your pixels-per-unit (PPU). Spawn entities from
a system that takes `Commands()`:

```typescript
import {
  defineSystem, Commands, Transform,
  RigidBody, BodyType, CircleCollider,
} from 'esengine';

const PPU = 100;

const spawnBall = defineSystem([Commands()], (cmds) => {
  cmds.spawn()
    .insert(Transform, { position: { x: 0, y: 250, z: 0 } })
    .insert(RigidBody, { bodyType: BodyType.Dynamic })
    .insert(CircleCollider, { radius: 24 / PPU, restitution: 0.4 });
});
```

- **`BodyType`** тАФ `Static` (immovable), `Kinematic` (moved by you), `Dynamic`
  (moved by the simulation).
- **Collider shapes** (from `esengine`) тАФ `BoxCollider`, `CircleCollider`,
  `CapsuleCollider`, `SegmentCollider`, `PolygonCollider`, `ChainCollider`. They
  share `density`, `friction`, `restitution`, `isSensor`, `categoryBits`,
  `maskBits`.
- **Joints** тАФ `RevoluteJoint`, `DistanceJoint`, `PrismaticJoint`, `WeldJoint`,
  `WheelJoint`.

## Collision layers

`categoryBits` (what layer a body is on) and `maskBits` (what it collides with)
are bit filters. Two bodies collide only if each one's category is in the
other's mask:

```typescript
const LAYER_PLAYER = 0x0001;
const LAYER_GROUND = 0x0008;

.insert(CircleCollider, {
  radius: 16 / PPU,
  categoryBits: LAYER_PLAYER,
  maskBits: LAYER_PLAYER | LAYER_GROUND,
});
```

## Forces and velocity

Read the `PhysicsAPI` resource in a system to push bodies around:

```typescript
import { defineSystem, Res } from 'esengine';
import { PhysicsAPI } from 'esengine/physics';

const push = defineSystem([Res(PhysicsAPI)], (physics) => {
  physics.applyImpulse(entity, { x: 0, y: 5 });        // jump
  physics.setLinearVelocity(entity, { x: 3, y: 0 });
  physics.applyTorque(entity, 2);
});
```

`PhysicsAPI` also exposes `applyForce`, `getLinearVelocity`, `setGravity` /
`getGravity`, angular variants, and spatial queries (raycast / overlap) that take
pixel-space arguments and scale by the live PPU.

## Collision events

The `PhysicsEvents` resource carries this frame's contacts:

```typescript
import { defineSystem, Res } from 'esengine';
import { PhysicsEvents } from 'esengine/physics';

const onHit = defineSystem([Res(PhysicsEvents)], (events) => {
  for (const c of events.collisionEnters) {
    // c.entityA, c.entityB, c.normalX, c.normalY, c.contactX, c.contactY
  }
  for (const s of events.sensorEnters) {
    // s.sensorEntity, s.visitorEntity
  }
});
```

<Aside type="caution">
  Set `isSensor: true` on a collider to get sensor enter/exit events **without**
  a physical response (trigger zones). Physics steps at a fixed timestep, not the
  frame delta, so read `PhysicsEvents` every frame.
</Aside>
