Estella has built-in [Spine](https://esotericsoftware.com/) skeletal animation,
driven by the `SpineAnimation` component and controlled at runtime through the
`Spine` resource.

## The `SpineAnimation` component

Add it in the editor and point it at your Spine assets. Its fields (import the
component from `esengine`):

- `skeletonPath`, `atlasPath` — the skeleton and atlas to load.
- `animation`, `loop`, `playing` — the current track and its playback state.
- `skin`, `timeScale`, `skeletonScale`, `flipX`, `flipY`, `color`, `layer`.

## Control at runtime with `Res(Spine)`

The `Spine` resource (from the `esengine/spine` subpath) is the imperative API:

```typescript
import { defineSystem, Query, Res, SpineAnimation } from 'esengine';
import { Spine } from 'esengine/spine';

const control = defineSystem([Query(SpineAnimation), Res(Spine)], (q, spine) => {
  for (const [entity] of q) {
    spine.setAnimation(entity, 'walk', true);      // play looped 'walk'
    spine.setSkin(entity, 'armor');                // swap skin
    spine.setMixDuration(entity, 'idle', 'walk', 0.2);  // 0.2s crossfade
  }
});
```

Methods include `setAnimation`, `setSkin`, `setMixDuration`, `setDefaultMix`,
`setTrackAlpha`, `setAttachment`, and the queries `getAnimations`, `getSkins`,
`getBounds`. You can also just write the `SpineAnimation` component's fields from
a `Query(Mut(SpineAnimation))` system for simple play/stop.

## Animation events

The `SpineEvents` resource (also from `esengine/spine`) publishes track events
once per frame — read it in the same frame:

```typescript
import { defineSystem, Res } from 'esengine';
import { SpineEvents } from 'esengine/spine';

const onSpine = defineSystem([Res(SpineEvents)], (spineEvents) => {
  for (const e of spineEvents.events) {
    // e.type: 'start' | 'interrupt' | 'end' | 'complete' | 'event'
    if (e.type === 'complete') { /* e.entity, e.animationName */ }
  }
});
```

<Aside type="note">
  The `SpineAnimation` **component** is in the main `esengine` package, but the
  `Spine` and `SpineEvents` **runtime resources** live on the `esengine/spine`
  subpath — spine is an optional side-module (loaded on demand per skeleton
  version 3.8 / 4.1 / 4.2), kept off the base bundle. Entities sharing a
  skeleton + atlas share one ref-counted skeleton.
</Aside>

<Aside type="caution">
  The bundled Spine Runtimes are proprietary. Shipping a game that uses Spine
  requires a valid Spine license from Esoteric Software — separate from

</Aside>
