Estella has built-in [Spine](https://esotericsoftware.com/) skeletal animation,
driven by the `SpineAnimation` component.

## The `SpineAnimation` component

You usually add it in the editor and point it at your Spine assets. Its fields:

- `skeletonPath`, `atlasPath` — the skeleton and atlas to load.
- `animation`, `loop`, `playing` — the current track and its playback state.
- `skin`, `timeScale`, `skeletonScale`, `flipX`, `flipY`, `color`, `layer`.

## Control it at runtime

Because it's a normal component, you change playback by writing its fields from
a system — query it with `Mut`:

```typescript
import { defineSystem, Query, Mut, SpineAnimation } from 'esengine';

const play = defineSystem([Query(Mut(SpineAnimation))], (q) => {
  for (const [entity, spine] of q) {
    spine.animation = 'walk';
    spine.loop = true;
    spine.playing = true;
  }
});
```

## Animation events

The `SpineEvents` resource publishes track events once per frame — read it in
the same frame:

```typescript
import { defineSystem, Res, SpineEvents } from 'esengine';

const onSpine = defineSystem([Res(SpineEvents)], (spineEvents) => {
  for (const e of spineEvents.events) {
    // e.type: 'start' | 'interrupt' | 'end' | 'complete' | 'event'
    if (e.type === 'complete') { /* e.entity, e.animationName */ }
  }
});
```

<Aside type="note">
  Estella supports Spine **3.8 / 4.1 / 4.2** — each skeleton version loads its
  own WebAssembly backend on demand (pay-for-use). Entities that share the same
  skeleton + atlas share one loaded, ref-counted skeleton.
</Aside>

<Aside type="caution">
  The bundled Spine Runtimes are proprietary. Shipping a game that uses Spine
  requires a valid Spine license from Esoteric Software — separate from
  Estella's own license.
</Aside>
