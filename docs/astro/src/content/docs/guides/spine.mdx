Estella has built-in [Spine](https://esotericsoftware.com/) skeletal animation,
driven by the `SpineAnimation` component and controlled at runtime through the
`Spine` resource (from the `esengine/spine` subpath).

## The `SpineAnimation` component

Add it in the editor and point it at your Spine assets. Fields (the component is
in the main `esengine` package):

- `skeletonPath`, `atlasPath` ‚Äî the skeleton and atlas to load.
- `animation`, `loop`, `playing` ‚Äî the current track.
- `skin`, `timeScale`, `skeletonScale`, `flipX`, `flipY`, `color`, `layer`.

## Play and blend animations

```typescript
import { defineSystem, Query, Res, SpineAnimation } from 'esengine';
import { Spine } from 'esengine/spine';

const control = defineSystem([Query(SpineAnimation), Res(Spine)], (q, spine) => {
  for (const [entity] of q) {
    spine.setAnimation(entity, 'run', true);            // play, looped
    spine.setDefaultMix(entity, 0.15);                  // default crossfade
    spine.setMixDuration(entity, 'idle', 'run', 0.25);  // specific transition
    spine.setTrackAlpha(entity, 0, 1.0);                // blend a track in/out
  }
});
```

## Skins and attachments

```typescript
spine.setSkin(entity, 'armored');
spine.setAttachment(entity, 'weapon-slot', 'sword');   // swap a slot's attachment
```

## Query a skeleton

```typescript
const anims  = spine.getAnimations(entity);   // string[]
const skins  = spine.getSkins(entity);        // string[]
const bounds = spine.getBounds(entity);       // { x, y, width, height } | null
```

`setEntityProps(entity, props)` updates several visual props (scale, flip, color,
time scale) at once.

## Animation events

The `SpineEvents` resource publishes track events once per frame ‚Äî read it in the
same frame:

```typescript
import { defineSystem, Res } from 'esengine';
import { SpineEvents } from 'esengine/spine';

const onSpine = defineSystem([Res(SpineEvents)], (spineEvents) => {
  for (const e of spineEvents.events) {
    // e.type: 'start' | 'interrupt' | 'end' | 'complete' | 'event'
    // e.entity, e.track, e.animationName; for 'event': e.eventName + values
  }
});
```

<Aside type="note">
  Estella supports Spine **3.8 / 4.1 / 4.2** ‚Äî each skeleton version loads its own
  WebAssembly backend on demand (pay-for-use), which is why the `Spine` and
  `SpineEvents` resources live on the `esengine/spine` subpath. Entities sharing a
  skeleton + atlas share one ref-counted skeleton.
</Aside>

<Aside type="caution">
