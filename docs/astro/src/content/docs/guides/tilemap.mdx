Estella renders tile layers in the C++/WebAssembly core. There are two ways in:
a high-level component that loads a whole map from a file, and a low-level API
for building and editing layers yourself.

## High-level: load a Tiled map

The `Tilemap` component points an entity at a Tiled JSON (`.tmj`) file via its
one field, `source`. You usually add it in the editor and pick the file; the
runtime then loads the map and spawns a `TilemapLayer` per layer. To do it
procedurally, spawn from a system that takes `Commands()`:

```typescript
import { defineSystem, Commands, Tilemap } from 'esengine';

const loadLevel = defineSystem([Commands()], (cmds) => {
  cmds.spawn().insert(Tilemap, { source: 'assets/maps/level.tmj' });
});
```

## Low-level: build layers with `TilemapAPI`

`TilemapAPI` is a singleton you import from `esengine` and call directly with a
layer entity. Spawn an entity, initialise a layer on it, then write tiles:

```typescript
import { defineSystem, Commands, Transform, TilemapAPI } from 'esengine';

const buildLayer = defineSystem([Commands()], (cmds) => {
  const layer = cmds
    .spawn()
    .insert(Transform, { position: { x: 0, y: 0, z: 0 } })
    .id();                                    // the layer entity

  TilemapAPI.initLayer(layer, 32, 32, 16, 16);   // 32x32 tiles, 16px cells

  const tiles = new Uint16Array(32 * 32);         // tile ids (0 = empty)
  tiles[0] = 1;
  TilemapAPI.setTiles(layer, tiles);

  TilemapAPI.setTile(layer, 5, 3, 7);             // one tile
  TilemapAPI.flipTile(layer, 5, 3, true, false, false);  // flip horizontally

  TilemapAPI.setTileAnimation(layer, 10, [        // animate tile id 10
    { tileId: 10, duration: 100 },
    { tileId: 11, duration: 100 },
  ]);
});
```

Other `TilemapAPI` methods: `getTile`, `fillRect`, `rotateTile`,
`initInfiniteLayer` (chunked infinite maps), `setTilesets`, `setTint`,
`setVisible`, and `tileToWorld` / `worldToTile`.

<Aside type="note">
  Tile **collision is play-mode only** — static colliders are spawned from
  collidable tiles (marked in the `.estileset` tileset) when the scene runs; they
  are never in the edit world or serialized. Tile animations advance
  automatically once set — you call `setTileAnimation` once.
</Aside>
