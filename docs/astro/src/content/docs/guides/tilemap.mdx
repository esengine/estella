Estella renders tile layers in the C++/WebAssembly core. Load a whole map from a
Tiled file, or build and edit layers yourself with the `Tilemaps` resource —
including multi-tileset layers, per-tile animation, and infinite chunked maps.

## High-level: load a Tiled map

The `Tilemap` component points an entity at a Tiled JSON (`.tmj`) file via its
one field, `source`. You usually add it in the editor; the runtime loads the map
and spawns a `TilemapLayer` per layer (with `cellSize`, tileset texture,
`renderLayer`, `tintColor`, `opacity`, `parallaxFactor`). In code:

```typescript
import { defineSystem, Commands, Tilemap } from 'esengine';

const loadLevel = defineSystem([Commands()], (cmds) => {
  cmds.spawn().insert(Tilemap, { source: 'assets/maps/level.tmj' });
});
```

## Build a layer in code

`Tilemaps` is the runtime API. Spawn an entity, initialise a layer, then fill it —
tile ids are `1`-based (`0` = empty):

```typescript
import { defineSystem, Commands, Res, Transform, Tilemaps } from 'esengine';

const build = defineSystem([Commands(), Res(Tilemaps)], (cmds, tilemap) => {
  const layer = cmds.spawn().insert(Transform, { position: { x: 0, y: 0, z: 0 } }).id();

  tilemap.initLayer(layer, 32, 32, 16, 16);    // 32×32 tiles, 16px cells
  tilemap.setTiles(layer, new Uint16Array(32 * 32));   // bulk load
  tilemap.setTile(layer, 5, 3, 7);             // one tile
  tilemap.fillRect(layer, 0, 0, 32, 1, 2);     // a row of tile 2
});
```

## Edit and transform tiles

```typescript
tilemap.getTile(layer, 5, 3);                       // read a tile id
tilemap.flipTile(layer, 5, 3, true, false, false);  // flipH, flipV, flipDiagonal
tilemap.rotateTile(layer, 5, 3, 90);                // degrees
tilemap.tileToWorld(layer, 5, 3, 0, 0);             // → { x, y }
tilemap.worldToTile(layer, wx, wy, 0, 0);           // → { x, y }
```

## Multiple tilesets

One layer can draw from several tilesets. Give it a slot table — each slot maps a
`firstId` range to a texture and its column count:

```typescript
tilemap.setTilesets(layer, [
  { firstId: 1,   textureHandle: grassTex, columns: 8 },
  { firstId: 256, textureHandle: propsTex, columns: 8 },
]);
```

## Tile animation

Set an animation on a tile id once — the plugin advances it every frame:

```typescript
tilemap.setTileAnimation(layer, 10, [
  { tileId: 10, duration: 100 },
  { tileId: 11, duration: 100 },
]);
```

## Infinite (chunked) maps

For large or streaming worlds, use an infinite layer and write chunk by chunk:

```typescript
tilemap.initInfiniteLayer(layer, 16, 16);
tilemap.setChunkTiles(layer, chunkX, chunkY, tiles, 32, 32);
```

`exportChunks` / `importChunks` serialize the sparse chunk data.

<Aside type="note">
  Tile **collision is play-mode only** — static colliders are spawned from
  collidable tiles (marked in the `.estileset` tileset, which also carries
  per-tile animation and terrain/auto-tile data) when the scene runs; they are
  never in the edit world or serialized. Other methods: `setTint`, `setVisible`,
  `setRenderProps`, `rotateTile`.
</Aside>
