Estella's UI is a CSS-like flexbox system solved in the C++ core. Compose an
interface from UI components — in the editor or in code — and read interaction
state from your systems.

## UI components

- **`UINode`** — the layout box. Flexbox fields live on it directly: `width`,
  `height` (and min/max), `flexGrow`, `flexShrink`, `flexBasis`, `alignSelf`,
  `margin*`, and `inset*` for absolute positioning. Layout is recomputed each
  frame.
- **`Text`** — text rendered through an SDF glyph atlas (`content`, `fontSize`,
  `color`, `align`, `bold`, `italic`, stroke/shadow, `wordWrap`, `richText`).
- **`UIVisual`** — solid-color, textured, sliced, or tiled fills.
- **`UIMask`** — clips its children.
- **`Interactable`** — marks an entity hit-testable (`enabled`, `blockRaycast`,
  `raycastTarget`).

## Build UI in code

The composition helpers are public. Spawn a node with `spawnUIEntity` +
`buildUINode`, sized with the dimension helpers `px` / `percent` / `auto`:

```typescript
import {
  defineSystem, GetWorld,
  spawnUIEntity, buildUINode, px, percent,
} from 'esengine';

const buildHUD = defineSystem([GetWorld()], (world) => {
  spawnUIEntity({
    world,
    node: buildUINode({ width: px(220), height: percent(100) }),
    text: { content: 'Score: 0', fontSize: 18 },
  });
});
```

Use **`FlexContainer`** (with `FlexDirection` / `JustifyContent` / `AlignItems`)
for flex layout, and the widget factories — **`createButton`**, `createToggle`,
`createSlider`, `createProgress`, `createDialog`, `createDropdown` — for
ready-made controls.

## Handling interaction

The hit-test system writes per-frame pointer state into `UIInteraction`
(`hovered`, `pressed`, `justPressed`, `justReleased`). It's **read-only for your
logic** — query it alongside `Interactable` to react:

```typescript
import { defineSystem, Query, Interactable, UIInteraction } from 'esengine';

const handleClicks = defineSystem(
  [Query(Interactable, UIInteraction)],
  (q) => {
    for (const [entity, _gate, state] of q) {
      if (state.justReleased) {
        // entity was clicked
      }
    }
  }
);
```

For richer input — bubbling `click` / `hover_enter` / `change` / `submit` /
`drag` events across the parent chain — read the `UIEvents` resource, the UI
event bus.

<Aside type="note">
  There is no `RectTransform`: `UINode` is real flexbox, sized with `px`,
  `percent`, and `auto`, plus `Absolute` positioning via insets. `UIInteraction`
  is written by the engine each frame — drive your logic from it or from
  `UIEvents`, don't mutate it.
</Aside>
