每个场景都通过一个相机来渲染——一个带 `Camera` 组件的实体。你可以控制缩放、跟随目标、
在屏幕与世界空间之间转换、震屏,以及在多个相机之间切换。

## `Camera` 组件

2D 游戏的关键字段:

- `orthoSize` —— 以世界单位计的垂直半高:**这就是你的缩放**(越小越放大)。
- `isActive` / `priority` —— 存在多个相机时哪个是活动相机(优先级最高的活动相机胜出)。
- `viewport`(一个 `Vec4`,`x/y/w/h` 取 0–1)—— 相机绘制到的屏幕矩形,用于分屏或画中画。
- `clearFlags`、`pixelPerfect`、`aspectRatio`。

## 跟随目标

给相机加一个 `FollowTarget` 并指向某个实体。通常在编辑器里设置;用代码:

```typescript
import { defineSystem, Query, Commands, Camera, FollowTarget } from 'esengine';

const follow = defineSystem([Query(Camera), Commands()], (cameras, cmds) => {
  for (const [cameraEntity] of cameras) {
    cmds.entity(cameraEntity).insert(FollowTarget, {
      target: playerEntity,   // 要跟随的实体(来自你的玩家 Query)
      offsetY: 40,
      deadzone: 24,           // 游走超过这个距离相机才移动
      damping: 0.25,          // 平滑(0 = 直接吸附)
    });
  }
});
```

## 屏幕 ↔ 世界

读 `CameraView` 转换坐标——用于把实体放到光标处,或判断什么在屏幕上:

```typescript
import { defineSystem, Res, CameraView, Input } from 'esengine';

const pick = defineSystem([Res(CameraView), Res(Input)], (view, input) => {
  const world  = view.screenToWorld(input.mouseX, input.mouseY);  // { x, y } | null
  const screen = view.worldToScreen(x, y);                        // { x, y } | null
  const mouse  = view.getWorldMousePosition();                    // { x, y } | null
  const bounds = view.getWorldBounds();                           // { left, right, bottom, top }
});
```

## 震屏

`shakeCamera` 给**渲染**视图加一个衰减扰动(不碰 Transform,所以总能恢复):

```typescript
import { shakeCamera } from 'esengine';

shakeCamera(app, { amplitude: 12, rotation: 0, frequency: 22, duration: 0.4 });
```

## 多相机

存在多个 `Camera` 实体时,`isActive` + `priority` 决定哪个生效。相机导演器可以在两个相机
之间随时间**混合**视图(而不是硬切)——用于电影化的镜头交接;`viewport` 则让两个相机
共享屏幕做分屏。

<Aside type="tip">
  用[补间](/docs/zh-cn/guides/animation/)(`TweenTarget.CameraOrthoSize`)动画化
  `orthoSize`,即可实现平滑的推近/拉远。
</Aside>
