Estella 的物理运行在 C++/WebAssembly 内核中(Box2D)。你给实体挂上 `RigidBody` 和一个
碰撞体,再通过物理资源来驱动或读取它。

## 启用物理

物理是一个可加载模块。在编辑器和 Web 运行时中,当场景用到物理组件时它会**自动加载**
——你无需接线。在自定义 App 里你也可以自己添加:

```typescript
import { PhysicsPlugin } from 'esengine/physics';

app.addPlugin(new PhysicsPlugin('physics.js', {
  gravity: { x: 0, y: -9.81 },
}));
```

## 刚体与碰撞体

挂上一个 `RigidBody` 加一个碰撞体。**碰撞体尺寸的单位是米,不是像素**——把像素尺寸
除以你的「每单位像素数」(PPU)。在带 `Commands()` 参数的系统里生成实体:

```typescript
import {
  defineSystem, Commands, Transform,
  RigidBody, BodyType, CircleCollider,
} from 'esengine';

const PPU = 100;

const spawnBall = defineSystem([Commands()], (cmds) => {
  cmds.spawn()
    .insert(Transform, { position: { x: 0, y: 250, z: 0 } })
    .insert(RigidBody, { bodyType: BodyType.Dynamic })
    .insert(CircleCollider, { radius: 24 / PPU, restitution: 0.4 });
});
```

- **`BodyType`** —— `Static`(不可移动)、`Kinematic`(由你移动)、`Dynamic`(由模拟
  驱动)。
- **碰撞体形状**(来自 `esengine`)—— `BoxCollider`、`CircleCollider`、
  `CapsuleCollider`、`SegmentCollider`、`PolygonCollider`、`ChainCollider`。它们共享
  `density`、`friction`、`restitution`、`isSensor`、`categoryBits`、`maskBits`。
- **关节** —— `RevoluteJoint`、`DistanceJoint`、`PrismaticJoint`、`WeldJoint`、
  `WheelJoint`。

## 碰撞层

`categoryBits`(刚体所在的层)与 `maskBits`(它会与哪些层碰撞)是位过滤器。两个刚体
只有在各自的 category 都落在对方的 mask 里时才会碰撞:

```typescript
const LAYER_PLAYER = 0x0001;
const LAYER_GROUND = 0x0008;

.insert(CircleCollider, {
  radius: 16 / PPU,
  categoryBits: LAYER_PLAYER,
  maskBits: LAYER_PLAYER | LAYER_GROUND,
});
```

## 力与速度

在系统里读取 `PhysicsAPI` 资源来推动刚体:

```typescript
import { defineSystem, Query, Res, RigidBody } from 'esengine';
import { PhysicsAPI } from 'esengine/physics';

const push = defineSystem([Query(RigidBody), Res(PhysicsAPI)], (q, physics) => {
  for (const [entity] of q) {          // 实际中可缩小到你的玩家组件
    physics.applyImpulse(entity, { x: 0, y: 5 });      // 跳跃
    physics.setLinearVelocity(entity, { x: 3, y: 0 });
    physics.applyTorque(entity, 2);
  }
});
```

`PhysicsAPI` 还提供 `applyForce`、`getLinearVelocity`、`setGravity` / `getGravity`、
角速度相关方法,以及空间查询(raycast / overlap)——查询接受像素单位参数,并按实时 PPU
缩放。

## 碰撞事件

`PhysicsEvents` 资源携带当前帧的接触信息:

```typescript
import { defineSystem, Res } from 'esengine';
import { PhysicsEvents } from 'esengine/physics';

const onHit = defineSystem([Res(PhysicsEvents)], (events) => {
  for (const c of events.collisionEnters) {
    // c.entityA、c.entityB、c.normalX、c.normalY、c.contactX、c.contactY
  }
  for (const s of events.sensorEnters) {
    // s.sensorEntity、s.visitorEntity
  }
});
```

<Aside type="caution">
  给碰撞体设 `isSensor: true`,可以在**没有物理响应**的情况下获得传感器进入/离开事件
  (触发区)。物理按固定时间步进,而不是帧间隔,所以每帧都要读 `PhysicsEvents`。
</Aside>
