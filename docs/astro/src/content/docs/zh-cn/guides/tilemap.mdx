Estella 在 C++/WebAssembly 内核中渲染瓦片图层。有两种用法:一个从文件加载整张地图的
高层组件,以及一套自己构建、编辑图层的低层 API。

## 高层:加载 Tiled 地图

`Tilemap` 组件通过它唯一的字段 `source`,把实体指向一个 Tiled JSON(`.tmj`)文件。
通常你在编辑器里添加该组件并选择文件;运行时随后加载地图,并为每个图层生成一个
`TilemapLayer`。要以代码方式做,在带 `Commands()` 参数的系统里生成:

```typescript
import { defineSystem, Commands, Tilemap } from 'esengine';

const loadLevel = defineSystem([Commands()], (cmds) => {
  cmds.spawn().insert(Tilemap, { source: 'assets/maps/level.tmj' });
});
```

## 低层:用 `Res(Tilemaps)` 构建图层

`Tilemaps` 资源就是瓦片地图运行时 API——用 `Res(Tilemaps)` 读取它,传入图层实体来
调用。先生成一个实体,在它上面初始化图层,再写入瓦片:

```typescript
import { defineSystem, Commands, Res, Transform, Tilemaps } from 'esengine';

const buildLayer = defineSystem([Commands(), Res(Tilemaps)], (cmds, tilemap) => {
  const layer = cmds
    .spawn()
    .insert(Transform, { position: { x: 0, y: 0, z: 0 } })
    .id();                                    // 图层实体

  tilemap.initLayer(layer, 32, 32, 16, 16);   // 32x32 格,16px 格子

  const tiles = new Uint16Array(32 * 32);      // 瓦片 id(0 = 空)
  tiles[0] = 1;
  tilemap.setTiles(layer, tiles);

  tilemap.setTile(layer, 5, 3, 7);             // 单个瓦片
  tilemap.flipTile(layer, 5, 3, true, false, false);   // 水平翻转

  tilemap.setTileAnimation(layer, 10, [        // 让瓦片 id 10 动起来
    { tileId: 10, duration: 100 },
    { tileId: 11, duration: 100 },
  ]);
});
```

其他方法:`getTile`、`fillRect`、`rotateTile`、`initInfiniteLayer`(分块的无限地图)、
`setTilesets`、`setTint`、`setVisible`,以及 `tileToWorld` / `worldToTile`。(同一个
对象也以 `TilemapAPI` 直接导出,供在系统之外使用。)

<Aside type="note">
  瓦片**碰撞仅在运行模式**存在——静态碰撞体在场景运行时由可碰撞瓦片(在 `.estileset`
  图集里标记)生成;它们不存在于编辑世界中,也不会被序列化。瓦片动画一旦设置便自动
  推进——`setTileAnimation` 只需调用一次。
</Aside>
