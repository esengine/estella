Estella 在 C++/WebAssembly 内核中渲染瓦片图层。你可以从 Tiled 文件加载整张地图,也可以
用 `Tilemaps` 资源自己构建、编辑图层——包括多图集图层、逐瓦片动画,以及无限分块地图。

## 高层:加载 Tiled 地图

`Tilemap` 组件通过唯一字段 `source` 把实体指向一个 Tiled JSON(`.tmj`)文件。通常你在
编辑器里添加它;运行时加载地图并为每个图层生成一个 `TilemapLayer`(带 `cellSize`、图集
贴图、`renderLayer`、`tintColor`、`opacity`、`parallaxFactor`)。用代码:

```typescript
import { defineSystem, Commands, Tilemap } from 'esengine';

const loadLevel = defineSystem([Commands()], (cmds) => {
  cmds.spawn().insert(Tilemap, { source: 'assets/maps/level.tmj' });
});
```

## 用代码构建图层

`Tilemaps` 是运行时 API。生成一个实体,初始化图层,再填充它——瓦片 id 从 `1` 开始
(`0` = 空):

```typescript
import { defineSystem, Commands, Res, Transform, Tilemaps } from 'esengine';

const build = defineSystem([Commands(), Res(Tilemaps)], (cmds, tilemap) => {
  const layer = cmds.spawn().insert(Transform, { position: { x: 0, y: 0, z: 0 } }).id();

  tilemap.initLayer(layer, 32, 32, 16, 16);    // 32×32 格,16px 格子
  tilemap.setTiles(layer, new Uint16Array(32 * 32));   // 批量载入
  tilemap.setTile(layer, 5, 3, 7);             // 单个瓦片
  tilemap.fillRect(layer, 0, 0, 32, 1, 2);     // 一行瓦片 2
});
```

## 编辑与变换瓦片

```typescript
tilemap.getTile(layer, 5, 3);                       // 读取瓦片 id
tilemap.flipTile(layer, 5, 3, true, false, false);  // 水平、垂直、对角翻转
tilemap.rotateTile(layer, 5, 3, 90);                // 度数
tilemap.tileToWorld(layer, 5, 3, 0, 0);             // → { x, y }
tilemap.worldToTile(layer, wx, wy, 0, 0);           // → { x, y }
```

## 多图集

一个图层可以从多个图集绘制。给它一张 slot 表——每个 slot 把一个 `firstId` 范围映射到
一张贴图及其列数:

```typescript
tilemap.setTilesets(layer, [
  { firstId: 1,   textureHandle: grassTex, columns: 8 },
  { firstId: 256, textureHandle: propsTex, columns: 8 },
]);
```

## 瓦片动画

给某个瓦片 id 设置一次动画——插件每帧推进它:

```typescript
tilemap.setTileAnimation(layer, 10, [
  { tileId: 10, duration: 100 },
  { tileId: 11, duration: 100 },
]);
```

## 无限(分块)地图

对于大型或流式世界,用无限图层并逐块写入:

```typescript
tilemap.initInfiniteLayer(layer, 16, 16);
tilemap.setChunkTiles(layer, chunkX, chunkY, tiles, 32, 32);
```

`exportChunks` / `importChunks` 序列化稀疏的分块数据。

<Aside type="note">
  瓦片**碰撞仅在运行模式**存在——静态碰撞体在场景运行时由可碰撞瓦片(在 `.estileset`
  图集里标记,该图集还携带逐瓦片动画和地形/自动图块数据)生成;它们不存在于编辑世界中,
  也不会被序列化。其他方法:`setTint`、`setVisible`、`setRenderProps`、`rotateTile`。
</Aside>
