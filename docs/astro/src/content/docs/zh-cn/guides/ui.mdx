Estella 的 UI 是在 C++ 内核里求解的 CSS 式弹性盒(flexbox)系统。你用 UI 组件组合界面——
在编辑器里或用代码——并在系统里处理交互。**没有 `RectTransform`**:`UINode` 就是真正的
flexbox,用 `px` / `percent` / `auto` 加 `Absolute` 内偏移(inset)来定尺寸。

## 组件

| 组件 | 职责 |
|---|---|
| `UINode` | 布局盒;持有逐项的 flex 字段(尺寸、grow/shrink、margin、inset)。 |
| `FlexContainer` | 用 flexbox 排布一个节点的子项(方向、对齐、gap、padding)。 |
| `Text` | 经 SDF 字形图集渲染文本(内容、字体、颜色、对齐、描边、阴影、富文本)。 |
| `UIVisual` | 填充——纯色、纹理、九宫格、平铺或填充。 |
| `UIMask` | 裁剪它的子项。 |
| `Interactable` | 标记节点可命中,从而能被悬停/点击。 |
| `UIInteraction` | 引擎每帧写入的指针状态(对你的逻辑只读)。 |

## 用代码搭建 UI

一个完整的暂停菜单——居中的列面板,带标题和一个按钮——用**启动系统**构建一次:

```typescript
import {
  defineSystem, addStartupSystem, GetWorld, Res, UIEvents,
  spawnUIEntity, createButton,
  FlexContainer, FlexDirection, JustifyContent, AlignItems, px,
} from 'esengine';

const buildPauseMenu = defineSystem([GetWorld(), Res(UIEvents)], (world, events) => {
  // A panel: 320×220 with a dark translucent background.
  const panel = spawnUIEntity({
    world,
    node: { width: px(320), height: px(220) },
    visual: { color: { r: 0.08, g: 0.10, b: 0.16, a: 0.95 } },
  });

  // Lay its children out as a centered vertical column.
  world.insert(panel, FlexContainer, {
    direction: FlexDirection.Column,
    justifyContent: JustifyContent.Center,
    alignItems: AlignItems.Center,
    gap: { x: 0, y: 16 },
  });

  // A title, parented to the panel.
  spawnUIEntity({ world, parent: panel, text: { content: 'Paused', fontSize: 28 } });

  // A button that resumes the game on click.
  createButton({
    world, events, parent: panel, text: 'Resume',
    node: { width: px(160), height: px(44) },
    states: {
      normal:  { color: { r: 0.20, g: 0.55, b: 1.0, a: 1 } },
      hover:   { color: { r: 0.30, g: 0.65, b: 1.0, a: 1 } },
      pressed: { color: { r: 0.12, g: 0.45, b: 0.9, a: 1 } },
    },
    onClick: () => { /* unpause, switch scene, hide the menu… */ },
  });
});

addStartupSystem(buildPauseMenu);
```

`spawnUIEntity({ world, parent?, node?, visual?, text? })` 返回实体;传 `parent` 即嵌套。
尺寸用维度辅助 `px`、`percent`、`auto`。其它控件工厂用法相同:`createToggle`、`createSlider`、
`createProgress`、`createDialog`、`createDropdown`。

<Aside type="tip">
  在编辑器里你可以可视化地搭同一棵树:添加 `UINode` 实体、嵌套它们,并在检视面板里加
  `FlexContainer` / `Text` / `UIVisual` / `Interactable`。
</Aside>

## `UINode` —— 布局盒

逐项的尺寸与放置在这里(维度用 `px()` / `percent()` / `auto()` 构造)。

| 属性 | 类型 | 默认 | 说明 |
|---|---|---|---|
| `position` | `UIPositionType` | `Relative` | `Relative`(在流中)或 `Absolute`(按 inset 放置)。 |
| `width` / `height` | Dimension | `auto` | 盒尺寸;`auto` = 由内容/flex 决定。 |
| `minWidth` / `minHeight` | Dimension | `auto` | 尺寸下界。 |
| `maxWidth` / `maxHeight` | Dimension | `auto` | 尺寸上界。 |
| `flexGrow` | number | `0` | 吸收多余空间的份额。 |
| `flexShrink` | number | `1` | 空间紧张时的收缩因子。 |
| `flexBasis` | Dimension | `auto` | grow/shrink 前的基准尺寸。 |
| `alignSelf` | `AlignSelf` | `Auto` | 逐项的交叉轴对齐覆盖。 |
| `marginLeft/Top/Right/Bottom` | Dimension | `0` | 外间距。 |
| `insetLeft/Top/Right/Bottom` | Dimension | `auto` | `Absolute` 时的边偏移。 |

## `FlexContainer` —— 布局

给节点加一个 `FlexContainer` 来排布它的子项。枚举与 CSS 对应。

| 属性 | 类型 | 默认 | 说明 |
|---|---|---|---|
| `direction` | `FlexDirection` | `Row` | `Row` / `Column` / `RowReverse` / `ColumnReverse`。 |
| `wrap` | `FlexWrap` | `NoWrap` | `NoWrap` / `Wrap`。 |
| `justifyContent` | `JustifyContent` | `Start` | 主轴分布(`Start`…`SpaceEvenly`)。 |
| `alignItems` | `AlignItems` | `Stretch` | 交叉轴对齐(`Start` / `Center` / `End` / `Stretch`)。 |
| `alignContent` | `AlignContent` | `Start` | 换行后各行的分布。 |
| `gap` | Vec2 | `{0, 0}` | 子项间距(x = 列间距,y = 行间距)。 |
| `padding` | Padding | `{0,0,0,0}` | 内间距(`left/top/right/bottom`)。 |

## `Text`

文本从 SDF 字形图集渲染,所以缩放清晰。

| 属性 | 类型 | 默认 | 说明 |
|---|---|---|---|
| `content` | string | `''` | 要显示的字符串。 |
| `fontFamily` | string | `'Arial'` | 字体族。 |
| `fontSize` | number | `24` | 字号(像素)。 |
| `color` | Color | `{1,1,1,1}` | 文本颜色。 |
| `align` | `TextAlign` | `Left` | 水平对齐。 |
| `verticalAlign` | `TextVerticalAlign` | `Top` | 垂直对齐。 |
| `wordWrap` | boolean | `true` | 在盒宽处换行。 |
| `overflow` | `TextOverflow` | `Visible` | 溢出处理。 |
| `lineHeight` | number | — | 行距倍率。 |
| `bold` / `italic` | boolean | `false` | 字形样式。 |
| `strokeColor` / `strokeWidth` | Color / number | — | 描边。 |
| `shadowColor` / `shadowBlur` / `shadowOffsetX` / `shadowOffsetY` | — | 投影。 |
| `richText` | boolean | `false` | 解析内联富文本标记。 |

## `UIVisual` —— 填充

`UIVisual` 绘制节点的背景。`visualType` 选择模式:

| `UIVisualType` | 使用 | 说明 |
|---|---|---|
| `None` | — | 不可见(仅布局/命中)。 |
| `SolidColor` | `color` | 着色方块。 |
| `Image` | `texture`、`uvOffset`、`uvScale` | 纹理方块 / 精灵区域。 |
| `NineSlice` | `sliceBorder` | 角固定、可缩放的面板。 |
| `Tiled` | `tileSize` | 纹理在盒内重复。 |
| `Filled` | `fillAmount`、`fillMethod`、`fillOrigin` | 裁剪填充(血条/进度条)。 |

## 处理交互

`createButton` 直接给你 `onClick`。对于自定义控件,命中系统会把每帧的指针状态写入任何带
`Interactable` 的实体上的 `UIInteraction`。它是**只读**的——查询它,绝不修改:

```typescript
import { defineSystem, Query, Interactable, UIInteraction } from 'esengine';

const onTiles = defineSystem([Query(Interactable, UIInteraction)], (q) => {
  for (const [entity, _gate, state] of q) {
    if (state.justReleased) { /* this node was clicked */ }
  }
});
```

| `UIInteraction` 字段 | 说明 |
|---|---|
| `hovered` | 指针在节点上方。 |
| `pressed` | 指针在节点上按住。 |
| `justPressed` | 本帧变为按下。 |
| `justReleased` | 本帧释放(一次点击)。 |

对于冒泡事件——`click`、`hover_enter`、`change`、`submit`、`drag` 沿父链向上——读
`UIEvents` 资源(UI 事件总线)。

## 最佳实践

- **用 flex,别用绝对定位**——用 `FlexContainer` + `flexGrow` 布局;`Absolute` inset 留给
  浮层和角标。
- **用 `percent` / `auto` 定尺寸**做分辨率无关;只在确实要固定尺寸处用硬 `px`。
- **逻辑由 `UIInteraction` 或 `UIEvents` 驱动**,绝不通过修改它们。
- **复用控件工厂**(`createButton`、`createSlider`……)获得一致的状态视觉,别手接悬停/按下。
- **九宫格面板**(`NineSlice`)让边框在任何尺寸都保持清晰。
