Estella 的 UI 是一套在 C++ 内核里求解的类 CSS 弹性盒(flexbox)系统。你在编辑器里用
UI 组件组合界面,再从系统里读取交互状态。

## UI 组件

在编辑器里把这些加到实体上来搭建界面:

- **`UINode`** —— 布局盒。弹性盒字段直接在它身上:`width`、`height`(以及 min/max)、
  `flexGrow`、`flexShrink`、`flexBasis`、`alignSelf`、`margin*`,以及用于绝对定位的
  `inset*`。布局每帧重新计算。
- **`Text`** —— 通过 SDF 字形图集渲染的文本(`content`、`fontSize`、`color`、`align`、
  `bold`、`italic`、描边/阴影、`wordWrap`、`richText`)。
- **`UIVisual`** —— 纯色、贴图、九宫格或平铺填充。
- **`UIMask`** —— 裁剪其子元素。
- **`Interactable`** —— 标记实体可命中测试(`enabled`、`blockRaycast`、
  `raycastTarget`)。

## 处理交互

命中测试系统每帧把指针状态写入 `UIInteraction`(`hovered`、`pressed`、`justPressed`、
`justReleased`)。它对你的逻辑而言是**只读的**——与 `Interactable` 一起查询即可响应:

```typescript
import { defineSystem, Query, Interactable, UIInteraction } from 'esengine';

const handleClicks = defineSystem(
  [Query(Interactable, UIInteraction)],
  (q) => {
    for (const [entity, _gate, state] of q) {
      if (state.justReleased) {
        // 该实体被点击了
      }
    }
  }
);
```

若需要更丰富的输入——沿父链冒泡的 `click` / `hover_enter` / `change` / `submit` /
`drag` 等事件——请读取 `UIEvents` 资源,即 UI 事件总线。

<Aside type="note">
  这里没有 `RectTransform`:`UINode` 是真正的弹性盒,用 `px`、`percent`、`auto` 尺寸
  (在检视面板里设置)加上通过 inset 的 `Absolute` 定位。`UIInteraction` 由引擎每帧
  写入——请用它或 `UIEvents` 驱动逻辑,不要去修改它。
</Aside>
