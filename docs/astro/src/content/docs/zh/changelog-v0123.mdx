## 打包 Entity

`Entity` 现在是 32 位打包结构体（20 位索引 + 12 位代数），取代原来的 `u32` 别名。在保持相同内存占用的前提下提供基于代数的生命周期校验。

- `Entity::make(index, generation)` — 从索引和代数构造
- `entity.index()` / `entity.generation()` / `entity.isValid()` — 查询
- `entity.id()` / `Entity::fromRaw(u32)` — FFI 序列化，用于 C++/JS 边界
- 支持 `operator<<` 日志输出
- 所有 Web 绑定层已适配类型化转换，不再使用裸类型转换

## World 上下文

新增 `World` 结构体，将 `Registry`、`ServiceRegistry` 和 `deltaTime` 打包为统一对象，传递给所有 `System::update()` 调用：

```cpp
void update(World& world) override {
    auto& registry = world.registry;
    auto& input = world.require<Input>();
    f32 dt = world.deltaTime;
}
```

消除了引擎系统中的全局状态访问模式。

## ServiceRegistry

集中式服务定位器，取代 `EngineContext` 中分散的单例访问器。系统通过 `World::require<T>()` 访问服务，而非直接操作全局状态。

## GfxDevice 图形抽象层

新增 `GfxDevice` 接口，将所有 OpenGL 调用隔离在 `GLDevice` 实现中：

- 强类型 `GfxEnums` 取代原始 GL 常量（`GfxBlendFactor`、`GfxDataType`、`GfxTextureFormat` 等）
- `SharedHandle` 提供纹理、着色器和帧缓冲的引用计数资源管理
- 为未来切换图形后端（Vulkan、WebGPU）铺平道路，无需改动渲染逻辑

## Manifest 驱动的工具链同步

用基于哈希的全量同步取代了脆弱的增量文件同步，由 `toolchain.manifest.json` 驱动：

- 计算 manifest 声明的所有文件的内容哈希（引擎源码、cmake 模块、第三方库）
- 哈希变化时清空并全量拷贝工具链目录，彻底消除残留文件问题
- 构建缓存 key 包含引擎源码路径，防止 debug/release 缓存冲突
- `compute_source_hash` 覆盖整个 `engine-src/` 目录，确保缓存失效可靠

## 实时构建进度

WASM 编译进度实时流式传输到编辑器构建面板：

- 解析 Ninja 输出（`[42/100] Building CXX object ...`）提取文件数和进度百分比
- 编译阶段切换（配置 → 编译 → 优化）同步更新进度条
- 构建日志面板实时滚动输出
- 错误报告合并 stdout + stderr（最后 30 行），编译错误不再被静默丢失

## 工具链修复

工具链面板新增"修复"按钮，仅下载缺失的工具（如 ninja），无需重新安装整个 emsdk 包。`package-emsdk.js` 打包脚本也会自动下载 ninja。
